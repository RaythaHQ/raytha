Audit Logs

Track all administrative activity in your Raytha installation. Monitor who did what and when for security and troubleshooting.

Accessing Audit Logs

Navigate to Settings > Audit Logs.

What's Logged

Audit logs capture administrative actions including:

  • Content: Creating, editing, publishing, deleting content items
  • Users: Creating, modifying, deleting users and admins
  • Settings: Configuration changes
  • Templates: Template modifications
  • Authentication: Login attempts and password resets
  • Site Pages: Page creation and widget changes

Log Entry Details

Each log entry includes:

Timestamp When the action occurred
User Email Who performed the action
Category Type of action (e.g., ContentItems, Users)
IP Address Origin of the request
Request Details Specific action and parameters

Filtering Logs

Use filters to find specific activity:

By Category

Filter by action type:

  • ContentItems
  • ContentTypes
  • Users
  • Administrators
  • Templates
  • Settings
  • SitePages

By Date Range

Select start and end dates to narrow results.

By User

Search by user email to see a specific admin's activity.

Viewing Request Details

Click on any log entry to see the full request payload, including:

  • Action performed
  • Entity ID affected
  • Before/after values (for updates)
  • Request parameters

Use Cases

Security Monitoring

  • Track failed login attempts
  • Monitor admin account changes
  • Identify suspicious activity patterns

Troubleshooting

  • Find who made a specific change
  • Determine when content was modified
  • Debug configuration issues

Compliance

  • Maintain activity records
  • Demonstrate access controls
  • Support audit requirements

Retention

Audit logs are retained indefinitely by default. For high-volume sites, consider implementing log archival through database maintenance.

Permissions

Only administrators with the Manage Audit Logs permission can view audit logs.
