SMTP Settings

Configure your email server settings to enable password resets, welcome emails, and other system notifications.

Accessing SMTP Settings

Navigate to Settings > SMTP.

Configuration Methods

SMTP can be configured two ways:

1. Environment Variables (Recommended)

Set SMTP credentials as server environment variables. This is more secure and allows different settings per environment.

2. Database Settings

Configure SMTP in the admin panel. Use the Override System Settings toggle to use database settings instead of environment variables.

SMTP Fields

Field Description Example
SMTP Host Mail server address smtp.sendgrid.net
SMTP Port Server port 587 (TLS) or 465 (SSL)
SMTP Username Authentication username apikey
SMTP Password Authentication password or API key (your API key)
Default From Address Sender email address noreply@example.com
Default From Name Sender display name My Website

Common Email Services

SendGrid

Host: smtp.sendgrid.net
Port: 587
Username: apikey
Password: (your SendGrid API key)

Mailgun

Host: smtp.mailgun.org
Port: 587
Username: (your Mailgun SMTP username)
Password: (your Mailgun SMTP password)

Amazon SES

Host: email-smtp.us-east-1.amazonaws.com
Port: 587
Username: (your SES SMTP username)
Password: (your SES SMTP password)

Gmail (Development Only)

Host: smtp.gmail.com
Port: 587
Username: (your email)
Password: (app password, not regular password)
Note: Gmail has sending limits and requires app passwords. Use a dedicated email service for production.

Override System Settings

When enabled, the database SMTP settings override any environment variables. This is useful for:

  • Testing different email configurations
  • Environments without access to server settings

Testing Email

To verify your SMTP configuration works:

  1. Create a test administrator account
  2. Check if the welcome email arrives
  3. Or use the forgot password flow

Troubleshooting

Emails not sending

  • Verify SMTP credentials are correct
  • Check firewall allows outbound connections on the SMTP port
  • Confirm the from address is verified with your email provider

Emails going to spam

  • Set up SPF and DKIM records for your domain
  • Use a reputable email service
  • Ensure the from address domain matches your website
