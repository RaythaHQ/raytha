Content Types

Content Types define the structure and organization of your content. Think of them as blueprints for your content items.

What is a Content Type?

A Content Type is a template that defines:

  • What fields are available (title, content, image, etc.)
  • How content items are organized
  • URL patterns for public pages

Examples: Blog Posts, Products, Events, Team Members, FAQs

Viewing Content Types

Navigate to any content type in the sidebar to see its items. Click the gear icon or go to Settings to manage the content type itself.

Creating a Content Type

  1. Go to Settings > Content Types
  2. Click Create Content Type
  3. Fill in the required fields:
    • Label (Singular): e.g., "Product"
    • Label (Plural): e.g., "Products"
    • Developer Name: URL-safe identifier (e.g., "products")
  4. Click Create

Content Type Settings

After creating a content type, configure these settings:

General Settings

  • Active: Enable/disable the content type
  • Description: Internal notes about the content type's purpose

Route Template

Define how URLs are generated for content items:

  • {PrimaryField} - The item's primary field value (usually title)
  • {CurrentYear} - Four-digit year
  • {CurrentMonth} - Two-digit month

Example: {CurrentYear}/{CurrentMonth}/{PrimaryField} creates URLs like /2024/03/my-article-title

Primary Field

The primary field is used as the main identifier for content items. It appears in:

  • Admin list views
  • URL generation
  • Search results

Set the primary field in the content type's configuration after adding fields.

Deleting a Content Type

Warning: Deleting a content type will move all its content items to the trash. This action affects your public website.

To delete: Go to the content type settings and click Delete. You'll be asked to confirm.
