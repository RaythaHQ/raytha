Views

Views are filtered, sorted lists of content items that can be displayed on your public website. Create multiple views to show different subsets of your content.

What is a View?

A View defines:

  • Which content items to display (filters)
  • How items are sorted
  • Which columns/fields to show
  • The public URL path
  • Pagination settings

Example: A "Recent Posts" view showing the 10 most recent published blog posts.

Managing Views

  1. Navigate to a content type
  2. Click Views in the submenu

Creating a View

  1. Click Create View
  2. Enter the view details:
    • Label: Display name (e.g., "All Posts")
    • Developer Name: URL path (e.g., "posts")
    • Description: Internal notes
  3. Click Create

Configuring Columns

Select which fields appear in the view:

  1. Go to the view's settings
  2. Check/uncheck fields under Columns
  3. Drag to reorder columns

Common columns include:

  • Primary field (title)
  • Creation time
  • Custom fields from your content type

Setting Filters

Filter which items appear in the view:

  1. Click Filter
  2. Add filter conditions:
    • Select a field
    • Choose an operator (equals, contains, greater than, etc.)
    • Enter the value
  3. Add multiple conditions with AND/OR logic

Example: Show only items where "Status equals Published" AND "Category equals News"

Sorting

Define the default sort order:

  1. Click Sort
  2. Select the field to sort by
  3. Choose ascending or descending
  4. Add additional sort fields if needed

Pagination Settings

  • Default Items Per Page: How many items show by default
  • Max Items Per Page: Maximum allowed via query parameter
  • Ignore Client Query Params: Prevent URL-based filtering/sorting

Publishing a View

Views must be published to appear on your website:

  1. Open the view
  2. Toggle Published to on
  3. Assign a web template to render the view

The view will be accessible at the URL path you specified (e.g., /posts).

Template Assignment

Each view needs a web template to render its content. Go to Settings in the view menu to select which template displays this view on the public website.
