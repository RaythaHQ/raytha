Roles & Permissions

Create roles to control what administrators can access. Assign system permissions and content type permissions to each role.

Accessing Roles

Navigate to Settings > Roles.

Built-in Roles

Role Description
Super Admin Full access to all features and content
Admin Full access to all features and content
Editor Manage content items and site pages

Creating a Role

  1. Click Create Role
  2. Enter:
    • Label: Display name (e.g., "Content Manager")
    • Developer Name: Identifier (e.g., "content_manager")
  3. Select permissions
  4. Click Create

System Permissions

Control access to admin features:

Permission Allows
Manage System Settings Edit site configuration, SMTP, authentication schemes
Manage Administrators Create, edit, and delete admin accounts
Manage Users Manage public user accounts and groups
Manage Content Types Create and configure content types and fields
Manage Templates Edit themes, web templates, and email templates
Manage Site Pages Create and edit site pages with widgets
Manage Audit Logs View activity and audit logs

Content Type Permissions

Control access to each content type separately:

Permission Allows
Read View content items in the admin
Edit Modify existing content items
Create Create new content items
Delete Delete content items

Example: A "Blog Editor" role might have full permissions for Posts but read-only for Products.

Editing a Role

  1. Click on the role name
  2. Modify permissions as needed
  3. Click Save

Assigning Roles to Administrators

Administrators can have multiple roles. Their effective permissions are the combination of all assigned roles.

To assign roles:

  1. Go to Settings > Administrators
  2. Edit the administrator
  3. Select roles from the list
  4. Click Save

Deleting a Role

  1. Open the role
  2. Click Delete
  3. Confirm deletion
Note: Built-in roles (Super Admin, Admin, Editor) cannot be deleted. Administrators with a deleted role lose those permissions.
